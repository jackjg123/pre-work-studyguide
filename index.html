HTML

  • HTML is used to create the structure of a webpage. It is made up of elements which work together to provide content
    and the basic framework for all pages on the web.
  • The following element types are commonly used to add content to pages: Text elements, Image elements,
    Header elements, List elements
  • Here are some common element types that are used to divide and describe a page: Meta-related elements, Content
    division elements, Semantic elements.
  • The head element contains information, or metadata, that helps the browser to render the page correctly. Elements
    included within the head are not visible when a user looks at a webpage.
  • The body element contains the content that is shown to the user. All the elements that we want the user to see and
    interact with must be placed inside this element.
  • An HTML attribute defines additional information about an element.
  • Metadata is data about our webpage that the browser uses to effectively serve our webpage to users.
  • The title element defines the title for a webpage.
  • Search engine optimization (SEO) is the process of improving the ranking of a website in search engines such as
    Google or Bing. Better SEO results in your website more frequently showing up in searches.
  • The style element defines the CSS styles associated with a website.
  • The base element defines the base URL for a webpage
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTML document.
  • The charset element sets our character encoding.
  • The http-equiv element sets an HTTP header for content.
  • The viewport element sets the default view for our webpage.
  • In semantic HTML elements, the name of the element describes its content and the role it plays on a webpage

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Internal style sheet contains CSS rules for the webpage in the head section of the HTML file.
  • The most common way to style a webpage is by using an external CSS style sheet.
  • Separation of concerns is a widely used term in web development and refers to separating code into different files based on use.
  • a selector defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • A margin property indicates how much space we want around the outside of an element.
  • The padding property adds space around the content inside an element.

Git

  • Git is a version control system that lets you manage and keep track of each version of your project. To access
    Git, we use the command line.
  • Git commands always start with git.
  • Using Git commands, we can initiate (or create) a repo, add our code to the repo, and view the different
    versions of our code, which are each stored in a unique commit.
  • Git is great for tracking changes locally on your own computer, but to collaborate with others, you need
    to use another tool, GitHub.
  • GitHub is a cloud-based hosting service that allows us to keep track of our projects inside repositories
    that are stored remotely.
  • Markdown is a lightweight plain-text formatting language.
  • GitHub issues are often formatted with a language called Markdown.
